- A custom table-cell renderer paired with a typed `meta.type` (e.g. `'text'|'number'|'date'|'status'`) on the same column can make the table engine dispatch to its built-in cell and silently discard the custom renderer — no error, wrong output. If the table library has this shape, `meta.type` must be `'custom'` or omitted whenever a custom `cell` is supplied.
- Paged fetchers behind a data-table component: confirm whether the table engine's page index is 0-based or 1-based (TanStack Table and similar are 0-based) BEFORE writing the slicing math. Off-by-one here only shows up when a user clicks page 2 — it passes `tsc`/eslint/unit tests silently. Contract: `start = page * pageSize`. Smoke-test pages 1→2→3 in the browser; the first row id must change every page.
- Generic `required` validators that do `!value` treat `0` as empty — any numeric enum/flag field starting at `0` needs an explicit `value !== null && value !== undefined` check instead of `required: true`.
- A component offering both a generic drag-reorder prop and a purpose-built "reorder" API on the same list/table: enabling both usually means one silently wins over the other. Enable exactly one interaction per concern.
- Native OS form-control chrome (date/time pickers, select arrows) inside a `display: flex` input wrapper can lose its default anchoring — CSS overrides targeting vendor pseudo-elements (`::-webkit-calendar-picker-indicator` etc.) frequently fail silently because the UA renders them in a shadow tree user CSS can't reliably reach. Prefer hiding the native pseudo (`opacity: 0` + `position: absolute` + `pointer-events: none`, not `display: none` — that can disable the control) and rendering a custom icon in the component's own suffix/adornment slot instead of fighting the UA stylesheet.
- User-generated markdown/HTML MUST be sanitized server-side before caching or rendering — comments and rich-text bodies are the classic XSS path.
