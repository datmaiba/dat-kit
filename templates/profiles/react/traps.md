- A custom table-cell renderer paired with a typed `meta.type` (e.g. `'text'|'number'|'date'|'status'`) on the same column can make the table engine dispatch to its built-in cell and silently discard the custom renderer ‚Äî no error, wrong output. If the table library has this shape, `meta.type` must be `'custom'` or omitted whenever a custom `cell` is supplied.
- Paged fetchers behind a data-table component: confirm whether the table engine's page index is 0-based or 1-based (TanStack Table and similar are 0-based) BEFORE writing the slicing math. Off-by-one here only shows up when a user clicks page 2 ‚Äî it passes `tsc`/eslint/unit tests silently. Contract: `start = page * pageSize`. Smoke-test pages 1‚Üí2‚Üí3 in the browser; the first row id must change every page.
- Generic `required` validators that do `!value` treat `0` as empty ‚Äî any numeric enum/flag field starting at `0` needs an explicit `value !== null && value !== undefined` check instead of `required: true`.
- A component offering both a generic drag-reorder prop and a purpose-built "reorder" API on the same list/table: enabling both usually means one silently wins over the other. Enable exactly one interaction per concern.
- Native OS form-control chrome (date/time pickers, select arrows) inside a `display: flex` input wrapper can lose its default anchoring ‚Äî CSS overrides targeting vendor pseudo-elements (`::-webkit-calendar-picker-indicator` etc.) frequently fail silently because the UA renders them in a shadow tree user CSS can't reliably reach. Prefer hiding the native pseudo (`opacity: 0` + `position: absolute` + `pointer-events: none`, not `display: none` ‚Äî that can disable the control) and rendering a custom icon in the component's own suffix/adornment slot instead of fighting the UA stylesheet.
- User-generated markdown/HTML MUST be sanitized server-side before caching or rendering ‚Äî comments and rich-text bodies are the classic XSS path.
- A green gate proves nothing until you've seen it fail ‚Äî after adding/changing a gate command, deliberately break something once and confirm it goes red. Classic instance: `tsc --noEmit` on a Vite solution tsconfig (`"files": []` + `references`) checks zero files and stays green forever; the real gate is `tsc -b`.
- vitest's Vite SSR transform turns named imports into property access ‚Äî a missing export becomes a silent `undefined` in tests but a hard `SyntaxError` in the real browser. Only `npm run build` (or a real browser) exercises true ESM semantics; under `verbatimModuleSyntax` a missing `import type` is exactly this class.
- `vite preview` (production-build server) does NOT use `server.proxy` from `vite.config.ts` ‚Äî declare a separate `preview.proxy` block if you test the built app against a real API.
- Vite does not hot-reload `vite.config.ts` changes (proxy rules etc.) ‚Äî restart the dev container/process after editing it, and check proxy rules cover EVERY static path the browser must reach, not just the API prefix.
