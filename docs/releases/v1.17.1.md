# dat-kit v1.17.1

This maintenance release removes an evidence-corruption path from the
scorecard helper before the open-platform branches are cut.

## Correction

- Historical `benchmarks/scorecard.jsonl` bytes are never rewritten during
  token attribution or reporting.
- `scripts/scorecard.py --append-record` validates and appends exactly one new
  schema-v2 record.
- Unsafe symlink, reparse-point, hard-linked, or swapped scorecard paths fail
  closed before any record bytes are written.
- Exact Claude session totals are persisted only when one transcript session
  maps to one scorecard task. Missing, overlapping, multi-task, and unsupported
  provider attribution remains `null` with a stable reason code.
- Report-time attribution is display-only and operates on an in-memory copy.
- Codex remains honest about its unverified transcript format and records
  `unsupported_provider` rather than estimating tokens.

## Compatibility and rollback

Existing schema-v1 and schema-v2 records remain byte-for-byte unchanged. The
new optional `token_attribution` object is ignored by the v1.17 reader, so no
history migration is required. Rollback to v1.17.0 changes only future writer
behavior; it must never rewrite or remove records appended by v1.17.1.
